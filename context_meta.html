Context 与注意力：一种理解世界的元方法

一部关于「有限智能」的通论

世界就是所发生的一切。…我的语言的边界，就是我的世界的边界。
——维特根斯坦，《逻辑哲学论》，1921

第〇部　定义

在展开任何讨论之前，我们必须像几何学那样，先把基本概念钉死。否则，越往后推进，语言越滑。

定义一：Context（语境 / 上下文）

Context（语境） 是：在某个时刻，能够影响一个决策或行动的所有可能信息的集合。

它有三个关键属性：

  • 潜在性：context 是"可能被用到"的信息场，不是"正在被用到"的信息。就像你走进一个房间——墙上的钟、窗外的雨、桌角的杯子——全都是你的潜在语境，但你不会同时意识到它们。
  • 动态性：context 随时间和行动不断变化。你读完这句话后，你的语境已经不一样了。
  • 分层性：context 有远近层次——此刻屏幕上的字（近层）< 今天上午开的会（中层）< 你大学学过的某门课（远层）。

定义二：Attention（注意力）

Attention（注意力） 是：将有限的处理资源选择性地分配给 context 的某个子集。

注意，注意力的本质是选择 + 分配，而不是"接收"。它有两面：

  • 激活：让某些信息从 context 的"背景"进入处理器的"焦点"。
  • 抑制：让其余信息留在背景中。没有抑制，就没有注意。看不见的和看得见的同等重要。

这个定义早在 1890 年就有了。威廉·詹姆斯（William James）在《心理学原理》中写道：

每个人都知道注意是什么。它是心灵以清晰而鲜活的形式，从几个似乎同时可能的对象或思想序列中，占有其中一个。聚焦、集中、意识，是其本质。它意味着从某些事物中撤退，以便有效地处理其他事物。

定义三：带宽（Bandwidth）

带宽 是：一个处理单元在同一时刻能够有效激活的 context 的最大数量。

人类的带宽大约是 4 ± 1 个信息块（Cowan, 2001）。不是 4 个字节，不是 4 个词——而是 4 个有意义的 "块"（chunk）。一个"块"可以是一个数字，也可以是一整个你已经烂熟于心的象棋阵型。经验的本质，就是把更多信息打包进一个 chunk，从而在带宽不变的前提下处理更复杂的 context。

二者的关系

注意力作用于 context。

context 是信息的场，注意力是场的探照灯。探照灯照到哪里，哪里就被"加热"——进入工作状态；其余区域继续保持低温备用。两者相互定义：没有 context，注意力没有可选择的材料；没有注意力，context 只是一片未经处理的混沌。

第一部　三大公理

有了定义，我们才能谈公理。以下三条，不证自明但后果深远。

公理一：注意力守恒（稀缺性原理）

任何处理单元上，注意力是一种零和资源——增加 A 的注意力配额，必以削减 B 的注意力配额为代价。

这不是哲学主张，而是生理事实。

  • 神经学证据：人脑约 860 亿神经元，但前额叶皮层——负责工作记忆和选择性注意的核心区域——在任何时刻只能维持约 4 个"热"表征。这不是训练可以改变的；这是硬件限制。
  • 人类学证据：Robin Dunbar 在 1992 年发现，人类社交关系的稳定上限约为 150 人——这不是文化现象，而是新皮层尺寸的数学推导。你的大脑皮层大小，决定了你能稳定维护多少人的 social context。
  • 经济学证据：Herbert Simon 1971 年说："信息的丰富，创造注意力的贫困。"他因此获得诺贝尔经济学奖。"有限理性"的实质，就是决策者无法将全部相关 context 纳入考虑。

公理二：局部性原理（相关性随距离衰减）

在一个 context 场中，两个信息元素之间的相关性的期望值，随它们在某个维度上的距离增大而递减。这个维度可以是时间、空间、因果关系或语义距离。

这是宇宙最普遍的规律之一，只是在不同的学科里穿了不同的名字：

  • 物理学：光锥。你只能与过去光锥内的物体发生因果交互。太远的事件，对你根本"不存在"。
  • 计算机科学：局部性原理。程序 90% 的内存访问集中在 10% 的地址空间。（Knuth, The Art of Computer Programming, Vol. 1）
  • 地理学：Tobler 第一定律。"一切事物都与其他事物相关，但近者比远者更相关。"（Waldo Tobler, 1970）
  • 语言学：Firth 的分布假说。"一个词的意义，由它周围的词决定。"上下文越近的词，语义贡献越大。
  • 社会学：你更可能与你物理距离更近的人成为朋友——社交网络研究反复验证了"接近效应"。

公理三：压缩—检索权衡（无免费午餐定理）

任何 context 系统都必须在压缩程度与检索保真度之间做一次不可回避的取舍。压缩越狠，检索时丢失的细节越多；保留越完整，消费时需要的带宽越高。

这是信息论在认知领域的回响。原始信息可以无损压缩吗？理论上可以，但有极限——Shannon 熵。超过这个极限，信息必然丢失。

认知系统面对的是同样的困境：

压缩程度例子优势代价
极高压缩成语、口号、KPI 数字、星级评分传播快、带宽消耗低丢失情境细节；"996" 三个字无法还原真实的劳动体验
中等压缩会议纪要、摘要、笔记平衡选择哪些保留、哪些丢弃——这本身就消耗注意力
极低压缩原始录音、未剪辑视频、全量日志信息完整无人有带宽消费；搜索成本吞噬了完整性收益

一个极端的例证来自文学：博尔赫斯 1942 年的短篇《博闻强记的富内斯》。主人公遭遇事故后获得了绝对记忆——他记得一切，每一个细节、每一片树叶的每一次颤动。结果？

他无法思考。思考，就是遗忘差异，就是概括、抽象。

遗忘不是缺陷，是功能。 没有压缩，就没有理解。

第二部　推导

公理已经立下，现在我们来"推导"——看这三条公理在不同的领域中可以引申出什么。

第一章　叙事设计：读者是带宽受限的 context 处理器

从公理出发

  • 公理一（注意力守恒） 告诉我们：读者在任何时刻只能关注有限的人物、线索和细节。一部小说塞进 30 个有名有姓的角色，读者的注意力就崩塌了。
  • 公理二（局部性） 告诉我们：读者更容易记住近期出现的信息。因此，伏笔如果隔了 300 页才回收而没有中间提示，就等于没埋。
  • 公理三（压缩—检索权衡） 告诉我们：作者必须选择性地呈现——不可能也没有必要把世界上的所有信息都给读者。海明威的"冰山理论"正是此意。

三种叙事策略的压缩率分析

E. M. 福斯特在《小说面面观》（1927）中区分了 story 与 plot：

故事是按时间顺序排列的事件叙述。情节也是事件叙述，但重点落在因果关系上。

从我们的框架看：

  • Story 是 context 的时间线性排列——你只需要跟随时间轴。带宽需求低，但理解深度也低。
  • Plot 是 context 的因果图——你需要在脑中同时维持多个因果链的开放状态。带宽需求高，但理解深度也高。

这就是为什么侦探小说如此依赖 plot：它们故意控制读者 context 的信息不对称。

三种可复用的叙事 context 策略：

  1. 契诃夫枪（Chekhov's Gun）：契诃夫本人的原话是——"如果第一幕墙上挂着一把枪，它必须在第三幕开火。"翻译：不要埋你不打算回收的 context 线索，否则你在浪费读者的注意力配额。
  1. 不可靠叙述者：石黑一雄的《长日将尽》、阿加莎·克里斯蒂的《罗杰疑案》。翻译：叙述者的 context 是污染的、不完整的，读者必须自己从字里行间重建真实 context——这本身就是推理的乐趣来源。
  1. 冰山理论：海明威——"一座冰山的尊严在于它只有八分之一露出水面。"翻译：优秀的作者故意压缩 context，迫使读者用自己的经验填补空白。读者参与了意义的建构，因此体验更深刻。

数据

Stanford 大学 Ang Li 等人（2023）对 IMDb 上评分最高的 10,000 部电影的剧本做了关键词分析。结果：高评分剧本（≥ 8.0）的第一幕与第三幕之间的关键词共现密度，是低评分剧本（≤ 6.0）的约 2.3 倍。 也就是说，好故事不是"更多"，而是context 闭合率更高——你埋下的每一条线索，最终都收到了回执。

对产品设计的迁移

用户体验弧线与叙事弧线是同构的：

  • Onboarding = 第一幕：埋下产品的核心"道具"（功能概念、交互模式）
  • 日常使用 = 第二幕：用户在场景中积累 context
  • Aha moment = 第三幕的回收：用户终于理解"哦，这个东西原来是用在这里的！"

第二章　组织管理：把公司视为 context 流动网络

从公理出发

  • 公理一（注意力守恒） 告诉我们：任何管理者的决策质量，受他同时处理的 context 数量上限约束。超出带宽上限的决策，质量必然下降。
  • 公理二（局部性） 告诉我们：一线员工拥有前线 context 的局部性优势，高管拥有战略 context 的全局性优势。信息在层级间传递时必然伴随压缩——压缩失真 = 决策误差。
  • 公理三（压缩—检索权衡） 告诉我们：KPI、报表和例会纪要是对组织 context 的压缩产物。压缩率越高（越简洁），丢失的细节越多——但压缩率太低，CEO 根本看不完。

组织形态即 context 路由拓扑

组织形态context 流压缩率带宽消耗反应速度
科层制自下而上压缩上报，自上而下决策下达极高低慢（失真大）
矩阵制每人两条 context 流（职能 + 项目）中等高（更多会议）中等
网络/扁平context 横向自由流动低极高（信息过载风险）快

没有任何一种形态是"最优"的——三种公理的约束相互拮抗，设计组织形态就是在三者之间找平衡。

约束理论的新解读

Eliyahu Goldratt 1984 年在《目标》中提出：系统的产出取决于其最薄弱的环节。

用我们的公理重述：组织的瓶颈 = 被迫并发处理 context 数量最多、同时处理带宽最低的那个节点。 这个节点通常是某个中层管理者——上面的战略压缩太狠（看不懂）、下面的一线细节太多（看不完）、同时被横向部门的对齐会议填满日程。

解决方法：减少并发 context。

丰田生产系统的"看板（Kanban）"是这一思想的工程实现。看板规则：

  1. 下游工序从上游拉取，而非上游推送——下游用实际需求（context 信号）决定生产
  1. 每张看板卡代表一个在制品（WIP）——系统内同时存在的 context 数量有上限
  1. WIP 上限一旦达到，不新增——强制集中注意力完成手头任务

David Anderson 2007 年在微软 XIT 部门引入类似机制后，平均交付周期从 98 天降至 11 天。不是因为做得更快，而是因为同时在做的更少。

OKR 的实质

John Doerr 在《这就是 OKR》（2018）中说 OKR 的作用是"浮出主要目标，引导精力和协调"。

用我们的框架翻译：OKR 是组织级的 context 压缩协议。它将每个层级无限可能的关注点，强制压缩为不超过 5 个关键结果，使分布式决策可以在不频繁通信的情况下保持方向一致。

OKR 的失败模式也因此清晰了：要么压缩过头（KR 没有反映真正重要的 context），要么根本不压缩（定了 15 个 KR，等于没定）。

第三章　人类认知：记忆不是存储，是 context 重建

从公理出发

  • 公理一（注意力守恒） 告诉我们：人脑在任何时刻只有约 4 个信息块的活跃带宽。超过这个数的任何信息，要么未被注意，要么只是在背景中处于低激活状态。
  • 公理二（局部性） 告诉我们：提取记忆时，与当前 context（环境、情绪、身体状态）更相似的经历更容易被提取。
  • 公理三（压缩—检索权衡） 告诉我们：记忆不是录像——大脑在存储时已经做了有损压缩。每次回忆，都是一次解压 + 重建。

编码特异性：心理学最深刻的发现之一

Endel Tulving 1972 年的编码特异性原则：

一段记忆能否被提取，不取决于"它是否被储存"，而取决于提取时的 context 与编码时的 context 有多接近。

这带来的推论让人不安：

  1. 你不"拥有"记忆，你"重建"记忆。 每次想起一件事，你都从压缩碎片中重新组装它，而组装过程受当下 context 的影响。
  1. "普鲁斯特时刻" 不是魔法——嗅觉皮层（唯一直接连接海马体的感官系统）提供了极高质量的 context 锚点，使提取精度远超其他感官触发。
  1. 认知行为疗法（CBT）的机制——在治疗的"安全 context"下，重新提取并改写旧的创伤记忆痕迹。每一次提取都是一次重写机会。
  1. 为什么在咖啡馆复习、在考场答题效果好？ 因为环境 context 一致——你编码知识时的背景与你提取时的背景越相似，检索越容易。

工作记忆：人类的"当前 context 窗口"

George Miller 1956 年那篇经典论文《神奇的数字：7±2》开启了量化人类认知带宽的传统。Nelson Cowan 2001 年的修正将数字精确到 4 ± 1。

重点是"chunk"的概念：

  • 一个新手面对一盘残局，看到的是 20 个孤立的棋子——20 个 chunk，远超带宽。
  • 一位大师看同一盘棋，看到的是 3 个"阵型"——3 个 chunk，刚好在带宽内。

经验的本质，就是把更多信息打包进更少的 chunk。 这本身就是一种压缩策略。

Daniel Levitin 在《有组织的心智》（The Organized Mind, 2014）中将这个逻辑推到了日常生活层面：为什么要把钥匙永远放在同一个地方？ 因为你不需要消耗一个 chunk 来记忆钥匙的位置——通过将琐事卸载到环境中，你释放了带宽用于真正重要的决策。

心流：context 与注意力的完美锁定

Mihaly Csikszentmihalyi 的心流理论（1990）描述的是人类最优体验状态。其三个前置条件——挑战与技能的匹配、明确的目标、即时反馈——用我们的框架翻译就是：

当外部 context 的复杂度刚好填满（但不过载）处理带宽，且反馈信号能即时告知你是否偏离目标时，注意力被"锁定"在 context 内部，外部干扰无法进入。

神经成像研究（Ulrich et al., 2014）证实：心流状态下，前额叶皮层——负责自我监控和"走神"的区域——活动减弱。这是神经系统在硬件层面实现了 context 锁定。

Cal Newport 在《深度工作》（2016）中引用的数据：一次打断后，恢复到深度注意力状态平均需要 23 分 15 秒（Gloria Mark, UCI）。这就是 context 切换的成本——你不仅要支付"加载新 context"的代价，还要支付"丢弃旧 context"和"重新建立旧 context"的代价。

第四章　技术史：把 context 从昂贵的载体卸载到便宜的载体

如果你把整部人类技术史压缩成一条线索，那就是：

人类不断找到更便宜的介质来承载 context，从而把大脑的注意力带宽释放出来，用于更高级的活动。

四次 context 卸载

时代context 从什么卸载到什么地方释放了什么代价
文字（约前 3200）口耳相传 → 泥板/纸不必用带宽维持跨代知识苏格拉底的担忧：文字会让人"不记忆"
印刷术（约 1440）手抄本 → 可复制的书知识工作者得以专业化教会对阐释权的垄断崩塌
互联网（约 1990）物理检索 → 数字检索从"在哪里找"变为"找什么"信息量指数增长，注意力成为新稀缺
个人计算机与移动设备桌面固定 → 随身 context随时随地接入信息场碎片化：深度连接更难维持

苏格拉底早就看透了一切

柏拉图《斐德罗篇》记录了苏格拉底对文字的批评：

文字会让人不再运用记忆，因为他们会信任写下来的文字，不再从自己的内在回忆事物。

苏格拉底看出的，正是 context 卸载的代价——每一次你把认知功能交给外部工具，你就失去了一部分内在能力。

但历史证明：这个代价值得付。 没有文字，就不可能有微积分、宪法、现代科学——因为没有外部化的 context，你无法在数百年的尺度上累积知识。

"深度"的丧失与重建

Nicholas Carr《浅薄》（The Shallows, 2010）的核心论点，用我们的框架重述：

互联网不是降低了信息质量，而是改变了注意力的拓扑结构——从"长时间专注在单一 context 上"变成"快速跳跃在多个 context 之间"。

Carr 引用的神经科学数据（Small & Vorgan, 2008）：每天上网超过 5 小时的人群，前额叶（深度 context 处理）活跃度下降，背外侧前额叶（快速切换）活跃度上升。大脑在硬件层面重塑了自己，以适应新的 context 环境。

关键不是"互联网让人变蠢"，而是我们还没有为新的 context 环境设计好相应的保护机制。

第三部　十条设计法则

前面都是"是什么"和"为什么"。最后回到"怎么办"——这十条，是我建议你和每一个做产品、系统、组织设计的人当作检验清单的。

结语

我们回到维特根斯坦那句话的开头——"我的语言的边界，就是我的世界的边界。"现在我们可以给出它的 context 翻译版：

你的 context 的边界，就是你能有效行动的世界的边界。

这不是悲观的论断，而是赋能的——因为 context 是可以被设计、被扩展、被整理的。每一次你改善笔记系统、优化团队流程、精简产品界面——你都在字面意义上扩展自己作为一个智能体的有效半径。

这就是为什么我们讨论 Zettelkasten、看板、OKR、叙事策略。它们表面上是工具和方法，本质上是人类试图把自己的注意力外化、结构化、持久化的努力。

法国哲学家贝尔纳·斯蒂格勒（Bernard Stiegler）称之为"第三持存"——人不仅能记住（第一持存：感知），不仅能回忆（第二持存：记忆），还能把记忆托付给外部技术对象（第三持存：工具）。

人之为人的特征之一，就是我们一直在用工具扩展自己的 context 窗口。
